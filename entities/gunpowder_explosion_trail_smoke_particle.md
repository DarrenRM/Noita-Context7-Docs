# Gunpowder Explosion Trail Smoke Particle

This entity defines the visual and physical properties of the smoke trail particles generated by gunpowder explosions in Noita. It dictates how these particles move, their appearance, and how they are emitted.

## Key Components and Attributes

### VelocityComponent
Controls the basic physics of the particle, including gravity and air resistance.

*   `gravity_y`: The strength of gravity pulling the particle downwards.
*   `air_friction`: How much air resistance affects the particle's movement.
*   `terminal_velocity`: The maximum speed the particle can reach due to gravity and friction.

### SimplePhysicsComponent
A basic physics component, often used in conjunction with `VelocityComponent` for simpler entities.

### SetStartVelocityComponent
Determines the initial speed and direction of the particles when they are spawned.

*   `randomize_speed.min`: The minimum initial speed.
*   `randomize_speed.max`: The maximum initial speed.
*   `randomize_angle.min`: The minimum initial angle (in radians).
*   `randomize_angle.max`: The maximum initial angle (in radians).

### DieIfSpeedBelowComponent
Causes the particle to despawn if its speed drops below a certain threshold.

*   `min_speed`: The minimum speed required for the particle to persist.

### SpriteComponent
Defines the visual representation of the particle itself.

*   `image_file`: The texture file used for the particle's sprite.
*   `additive`: Whether the sprite uses additive blending (makes colors brighter).
*   `emissive`: Whether the sprite emits light.
*   `alpha`: The transparency of the sprite.
*   `z_index`: Controls the rendering order of the sprite.

### SpriteParticleEmitterComponent (x2)
These components are responsible for emitting other particles, creating secondary visual effects.

#### First SpriteParticleEmitterComponent (fire_falling.xml)
This emitter likely contributes to the initial fiery aspect of the explosion trail.

*   `sprite_file`: The sprite file for the emitted particles.
*   `emission_interval_min_frames`, `emission_interval_max_frames`: Controls how often particles are emitted.
*   `count_min`, `count_max`: The number of particles emitted per interval.
*   `scale.x`, `scale.y`: The base scale of the emitted particles.
*   `lifetime`: How long the emitted particles last.
*   `randomize_scale.min/max_x/y`: Adds variation to the scale of emitted particles.
*   `randomize_velocity.min/max_y`: Adds vertical velocity variation to emitted particles.

#### Second SpriteParticleEmitterComponent (smoke_gunpowder_04.xml)
This emitter is specifically for emitting smoke particles.

*   `sprite_file`: The sprite file for the emitted smoke particles.
*   `emission_interval_min_frames`, `emission_interval_max_frames`: Controls how often smoke particles are emitted.
*   `count_min`, `count_max`: The number of smoke particles emitted per interval.
*   `scale.x`, `scale.y`: The base scale of the emitted smoke particles.
*   `sprite_random_rotation`: Whether the emitted smoke particles have random rotation.
*   `randomize_scale.min/max_x/y`: Adds variation to the scale of smoke particles.
*   `randomize_velocity.min/max_y`: Adds vertical velocity variation to smoke particles.
*   `randomize_velocity.min/max_x`: Adds horizontal velocity variation to smoke particles.

### ParticleEmitterComponent (x2)
These components emit actual game materials (like smoke and flame) as particles, rather than just sprites.

#### First ParticleEmitterComponent (smoke)
Emits "smoke" material particles.

*   `emitted_material_name`: The name of the material to emit.
*   `offset.x`, `offset.y`: The position offset from the entity's origin where particles are emitted.
*   `x_pos_offset_min/max`, `y_pos_offset_min/max`: Defines a rectangular area for particle emission.
*   `x_vel_min/max`, `y_vel_min/max`: The range of initial velocities for emitted particles.
*   `count_min`, `count_max`: The number of particles emitted per interval.
*   `lifetime_min`, `lifetime_max`: The duration each emitted particle exists.
*   `create_real_particles`: Whether to create actual game particles.
*   `emission_interval_min_frames`, `emission_interval_max_frames`: Controls the emission rate.

#### Second ParticleEmitterComponent (flame)
Emits "flame" material particles.

*   `emitted_material_name`: The name of the material to emit.
*   `offset.x`, `offset.y`: The position offset from the entity's origin.
*   `gravity.y`: The gravity applied to these flame particles.
*   `x_pos_offset_min/max`, `y_pos_offset_min/max`: Defines the emission area.
*   `x_vel_min/max`, `y_vel_min/max`: The range of initial velocities for emitted flame particles.
*   `count_min`, `count_max`: The number of flame particles emitted per interval.
*   `lifetime_min`, `lifetime_max`: The duration each emitted flame particle exists.
*   `create_real_particles`: Whether to create actual game particles.
*   `emission_interval_min_frames`, `emission_interval_max_frames`: Controls the emission rate.
