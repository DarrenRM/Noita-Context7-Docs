# Gunpowder Explosion Smoke Trail Particle

This entity defines the visual and physical properties of smoke particles generated by a gunpowder explosion in Noita. It's designed to create a trailing effect for projectiles or explosions.

## Key Components and Attributes

### VelocityComponent
Controls the particle's movement in the environment.

*   `gravity_y`: The strength of gravity pulling the particle downwards.
*   `air_friction`: Resistance to movement caused by air.
*   `terminal_velocity`: The maximum speed the particle can reach due to gravity and friction.

### SimplePhysicsComponent
Enables basic physics interactions for the particle.

### SetStartVelocityComponent
Determines the initial speed and direction of the particle upon creation.

*   `randomize_speed.min`: Minimum initial speed.
*   `randomize_speed.max`: Maximum initial speed.
*   `randomize_angle.min`: Minimum initial angle (in radians).
*   `randomize_angle.max`: Maximum initial angle (in radians).

### DieIfSpeedBelowComponent
Defines a condition for the particle to disappear.

*   `min_speed`: The particle will be destroyed if its speed drops below this value.

### SpriteComponent
Defines the visual appearance of the particle.

*   `image_file`: Path to the sprite image.
*   `additive`: Whether the sprite uses additive blending.
*   `emissive`: Whether the sprite emits light.
*   `alpha`: The transparency of the sprite.
*   `update_transform_rotation`: Whether the sprite's rotation should update with its movement.

### SpriteParticleEmitterComponent
Responsible for emitting other particles, specifically smoke sprites.

*   `sprite_file`: Pattern for the smoke sprite files (e.g., `smoke_gunpowder_0$[1-3].xml`).
*   `emission_interval_min_frames`, `emission_interval_max_frames`: Controls how often new sprites are emitted.
*   `count_min`, `count_max`: The number of sprites emitted per interval.
*   `scale.x`, `scale.y`: Base scale of the emitted sprites.
*   `sprite_random_rotation`: Whether emitted sprites should have random rotation.
*   `randomize_scale.min_x`, `randomize_scale.max_x`, `randomize_scale.min_y`, `randomize_scale.max_y`: Random variation in scale.
*   `randomize_velocity.min_y`, `randomize_velocity.max_y`, `randomize_velocity.min_x`, `randomize_velocity.max_x`: Random variation in the initial velocity of emitted sprites.

### ParticleEmitterComponent (Smoke)
Emits actual "smoke" material particles.

*   `emitted_material_name`: The material type of the particles to emit ("smoke").
*   `offset.x`, `offset.y`: Offset from the emitter's position.
*   `x_pos_offset_min`, `y_pos_offset_min`, `x_pos_offset_max`, `y_pos_offset_max`: Area where particles are spawned.
*   `x_vel_min`, `x_vel_max`, `y_vel_min`, `y_vel_max`: Initial velocity range for emitted particles.
*   `count_min`, `count_max`: Number of particles emitted per interval.
*   `lifetime_min`, `lifetime_max`: Duration of emitted particles.
*   `create_real_particles`: Whether to create actual game particles.
*   `emission_interval_min_frames`, `emission_interval_max_frames`: Controls emission frequency.
*   `is_emitting`: Whether the emitter is active.

### ParticleEmitterComponent (Flame)
Emits "flame" material particles, likely for a brief initial burst.

*   `emitted_material_name`: The material type of the particles to emit ("flame").
*   `offset.x`, `offset.y`: Offset from the emitter's position.
*   `gravity.y`: Gravity applied to these flame particles.
*   `x_pos_offset_min`, `y_pos_offset_min`, `x_pos_offset_max`, `y_pos_offset_max`: Spawn area.
*   `x_vel_min`, `x_vel_max`, `y_vel_min`, `y_vel_max`: Initial velocity range.
*   `count_min`, `count_max`: Number of particles emitted per interval.
*   `lifetime_min`, `lifetime_max`: Duration of emitted particles.
*   `create_real_particles`: Whether to create actual game particles.
*   `emission_interval_min_frames`, `emission_interval_max_frames`: Controls emission frequency.
*   `is_emitting`: Whether the emitter is active.
