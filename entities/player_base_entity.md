# Player Base Entity

This document details the core `player_base.xml` entity, defining the fundamental attributes and behaviors of the player character in Noita. This is a crucial file for understanding player mechanics and for modding player-related features.

## Core Components

The player entity is composed of numerous components, each responsible for a specific aspect of its functionality. Here are some of the most significant ones:

### `HotspotComponent`

Defines various points of interaction and reference for the player character.

| Tag           | Description                               |
|---------------|-------------------------------------------|
| `hand`        | Position for holding items/weapons.       |
| `shoot_pos`   | Origin point for shooting projectiles.    |
| `kick_pos`    | Origin point for kicking actions.         |
| `crouch_sensor`| Defines the player's position when crouched. |
| `right_arm_root`| The pivot point for the player's right arm. |
| `cape_root`   | The attachment point for the player's cape. |
| `belt_root`   | The attachment point for the player's belt. |

### `PlatformShooterPlayerComponent`

Manages player movement, camera behavior, and interaction with the environment.

| Attribute                         | Description                                                              |
|-----------------------------------|--------------------------------------------------------------------------|
| `center_camera_on_this_entity`    | If `1`, the camera will follow the player.                               |
| `aiming_reticle_distance_from_character` | How far the aiming reticle is from the player.                         |
| `camera_max_distance_from_character` | The maximum distance the camera can be from the player.                  |
| `move_camera_with_aim`            | If `1`, the camera will move with the player's aim.                      |
| `eating_area_min/max.x/y`         | Defines the area where the player can "eat" or interact with certain entities. |
| `eating_cells_per_frame`          | How many cells the player can consume per frame.                         |

### `CharacterDataComponent`

Contains fundamental character data, including collision, movement, and flight mechanics.

| Attribute                     | Description                                                                 |
|-------------------------------|-----------------------------------------------------------------------------|
| `platforming_type`            | Defines the type of platforming behavior (e.g., 2 for standard player).     |
| `check_collision_max_size_x/y`| Maximum size for collision checks.                                          |
| `climb_over_y`                | The vertical distance the player can climb over obstacles.                  |
| `collision_aabb_min/max_x/y`  | Axis-Aligned Bounding Box for collision detection.                          |
| `eff_hg_...`                  | Various parameters related to the "hit ground" effect (e.g., velocity, damage). |
| `fly_time_max`                | Maximum duration the player can fly.                                        |
| `fly_recharge_spd`            | Speed at which flight energy recharges when not on the ground.              |
| `fly_recharge_spd_ground`     | Speed at which flight energy recharges when on the ground.                  |
| `flying_needs_recharge`       | If `1`, flight requires energy recharge.                                    |
| `gravity`                     | The gravitational pull affecting the player (0 for no gravity in air).      |
| `buoyancy_check_offset_y`     | Vertical offset for buoyancy checks.                                        |

### `CharacterPlatformingComponent`

Governs the player's movement physics, including jumping, running, and flying.

| Attribute             | Description                                                              |
|-----------------------|--------------------------------------------------------------------------|
| `jump_velocity_y`     | The vertical force applied when jumping.                                 |
| `jump_velocity_x`     | The horizontal force applied when jumping.                               |
| `fly_speed_max_up`    | Maximum upward speed when flying.                                        |
| `fly_speed_max_down`  | Maximum downward speed when flying.                                      |
| `fly_speed_mult`      | Multiplier for flight speed.                                             |
| `mouse_look`          | If `1`, allows aiming with the mouse.                                    |
| `keyboard_look`       | If `1`, allows aiming with the keyboard.                                 |
| `pixel_gravity`       | The gravity applied to individual pixels (used for certain effects).     |
| `run_velocity`        | The player's running speed.                                              |
| `accel_x`             | Horizontal acceleration.                                                 |
| `velocity_min/max_x/y`| Minimum and maximum velocity limits for the player.                      |

### `GenomeDataComponent`

Defines the player's genetic traits and role in the ecosystem.

| Attribute         | Description                               |
|-------------------|-------------------------------------------|
| `herd_id`         | Identifier for the player's herd.         |
| `food_chain_rank` | The player's position in the food chain.  |
| `is_predator`     | If `1`, the player is a predator.         |

### `HitboxComponent`

Defines the collision boundaries for the player.

| Tag           | Description                                                              |
|---------------|--------------------------------------------------------------------------|
| (default)     | Standard hitbox for the player.                                          |
| `crouched`    | Hitbox when the player is crouched (often smaller).                      |
| `is_player`   | Marks this hitbox as belonging to the player.                            |

### `Inventory2Component`

Manages the player's inventory slots and organization.

| Attribute               | Description                               |
|-------------------------|-------------------------------------------|
| `quick_inventory_slots` | Number of slots for quick access.         |
| `full_inventory_slots_x`| Number of slots in the horizontal direction for the full inventory. |
| `full_inventory_slots_y`| Number of slots in the vertical direction for the full inventory.   |

### `SpriteComponent`

Handles the visual representation of the player and its various states.

| Tag                       | Description
