# Tentacles Enemy Entity

This document details the configuration for the "Tentacles" enemy entity in Noita, primarily focusing on its AI, damage, physics, and visual properties.

## Base Entity Configuration

The entity inherits from `base_enemy_flying.xml`, indicating it's a flying enemy with standard AI behaviors.

### AnimalAIComponent

Controls the enemy's artificial intelligence and combat capabilities.

| Attribute                 | Description                                                                 |
| :------------------------ | :-------------------------------------------------------------------------- |
| `attack_ranged_entity_file` | Specifies the projectile entity to be used for ranged attacks (e.g., `buckshot.xml`). |
| `attack_ranged_enabled`   | Enables ranged attacks (1 for enabled, 0 for disabled).                     |
| `attack_dash_enabled`     | Enables dash attacks (1 for enabled, 0 for disabled).                       |
| `attack_ranged_frames_between` | Sets the delay in frames between consecutive ranged attacks.                |
| `can_fly`                 | Allows the entity to fly (1 for enabled, 0 for disabled).                   |

### DamageModelComponent

Defines the entity's health and how it reacts to damage.

| Attribute           | Description                                                              |
| :------------------ | :----------------------------------------------------------------------- |
| `hp`                | The entity's hit points (e.g., `1.2`).                                   |
| `ragdoll_material`  | The material of the ragdoll when the entity is defeated (e.g., `acid`).  |
| `ragdoll_filenames_file` | Path to a file containing filenames for ragdoll parts.                   |

### PathFindingComponent

Configures the entity's pathfinding capabilities.

| Attribute | Description                               |
| :-------- | :---------------------------------------- |
| `can_fly` | Allows pathfinding while airborne (1/0). |

### SpriteComponent

Determines the visual appearance of the entity.

| Attribute   | Description                                  |
| :---------- | :------------------------------------------- |
| `image_file` | Path to the sprite's graphical definition. |
| `offset_x`  | Horizontal offset for the sprite.            |
| `offset_y`  | Vertical offset for the sprite.              |

### HitboxComponent

Defines the collision boundaries of the entity.

| Attribute   | Description                                                                                             |
| :---------- | :------------------------------------------------------------------------------------------------------ |
| `_enabled`  | Enables or disables the hitbox (1/0).                                                                   |
| `aabb_min_x`| Minimum X-coordinate of the Axis-Aligned Bounding Box.                                                  |
| `aabb_max_x`| Maximum X-coordinate of the Axis-Aligned Bounding Box.                                                  |
| `aabb_min_y`| Minimum Y-coordinate of the Axis-Aligned Bounding Box.                                                  |
| `aabb_max_y`| Maximum Y-coordinate of the Axis-Aligned Bounding Box.                                                  |
| `is_enemy`  | Marks the hitbox as belonging to an enemy (1/0).                                                        |
| `is_item`   | Marks the hitbox as belonging to an item (1/0).                                                         |
| `is_player` | Marks the hitbox as belonging to the player (1/0).                                                      |

### GenomeDataComponent

Provides genetic information, potentially for breeding or herd behavior.

| Attribute | Description                               |
| :-------- | :---------------------------------------- |
| `herd_id` | Identifier for the entity's herd (e.g., `slimes`). |

## Material Inventory Component

Manages the materials contained within the entity.

| Attribute                 | Description                                                                                             |
| :------------------------ | :------------------------------------------------------------------------------------------------------ |
| `_enabled`                | Enables or disables the component (1/0).                                                                |
| `drop_as_item`            | Determines if materials are dropped as items upon death (1/0).                                          |
| `leak_on_damage_percent`  | The percentage of damage at which materials start leaking from the entity.                              |
| `count_per_material_type` | A list of materials and their counts within the entity.                                                 |
| `Material material="acid"`| Specifies the material type (e.g., `acid`).                                                           |
| `count="200"`             | The quantity of the specified material.                                                                 |
