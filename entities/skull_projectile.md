# Skull Projectile

This document details the configuration of the Skull projectile entity in Noita, focusing on attributes relevant to AI-assisted modding.

## Core Entity Configuration

The `Entity` tag defines the base properties of the projectile.

*   **`name`**: `$projectile_default` (This is a placeholder, the actual name would be defined elsewhere or inherited).
*   **`tags`**: `projectile_player` (Indicates this projectile is fired by the player).

## Base Projectile Properties

The `<Base file="data/entities/base_projectile.xml">` tag inherits common projectile behaviors.

### `VelocityComponent`

Controls the physical movement of the projectile.

*   **`gravity_y`**: `0` (No vertical gravity applied).
*   **`air_friction`**: `0` (No air resistance).
*   **`mass`**: `0.1` (Light mass).

## Projectile Behavior (`ProjectileComponent`)

This is the primary component defining the projectile's unique characteristics.

*   **`_enabled`**: `1` (Component is active).
*   **`lob_min` / `lob_max`**: `0.8` / `1.0` (Controls the arc of the projectile, with minimal lobbing).
*   **`speed_min` / `speed_max`**: `100` / `120` (Defines the projectile's initial speed range).
*   **`die_on_low_velocity`**: `0` (Projectile does not disappear if its speed drops too low).
*   **`velocity_sets_y_flip`**: `1` (Vertical flipping of the sprite is tied to velocity).
*   **`on_death_explode`**: `1` (Projectile explodes upon death).
*   **`on_lifetime_out_explode`**: `1` (Projectile explodes when its lifetime expires).
*   **`damage`**: `0.05` (Base damage dealt by the projectile).
*   **`on_collision_die`**: `0` (Projectile does not die upon colliding with surfaces).
*   **`lifetime`**: `100` (Duration in frames before the projectile expires).
*   **`bounces_left`**: `99` (High number of bounces allowed).
*   **`bounce_always`**: `1` (Projectile will always bounce off surfaces).
*   **`knockback_force`**: `1.5` (Force applied to entities upon collision).

### `config_explosion`

Defines the properties of the explosion that occurs when the projectile dies.

*   **`never_cache`**: `1` (Prevents caching of this explosion effect).
*   **`camera_shake`**: `0` (No camera shake on explosion).
*   **`explosion_radius`**: `20` (Area of effect for the explosion).
*   **`explosion_sprite`**: `data/particles/explosion_032.xml` (Visual effect for the explosion).
*   **`create_cell_probability`**: `1` (Always creates a cell upon explosion).
*   **`create_cell_material`**: `bone` (The material of the created cell).
*   **`ray_energy`**: `100000` (High energy for potential ray interactions).
*   **`hole_enabled`**: `1` (Creates a hole in surfaces upon explosion).
*   **`damage`**: `0.5` (Damage dealt by the explosion itself).
*   **`damage_mortals`**: `1` (Explosion damages living entities).
*   **`physics_explosion_power.min` / `physics_explosion_power.max`**: `0.3` / `0.5` (Controls the physics force of the explosion).
*   **`sparks_enabled`**: `1` (Sparks are generated by the explosion).

## Homing Behavior (`HomingComponent`)

Enables the projectile to track targets.

*   **`target_tag`**: `homing_target` (The tag of entities to be targeted).
*   **`homing_targeting_coeff`**: `5` (Strength of the homing behavior).
*   **`detect_distance`**: `350` (Maximum distance at which targets can be detected).
*   **`homing_velocity_multiplier`**: `1.0` (Multiplier for velocity when homing).

## Visual Representation (`SpriteComponent`)

Defines the visual appearance of the projectile.

*   **`_enabled`**: `1` (Component is active).
*   **`image_file`**: `data/projectiles_gfx/skull.xml` (Path to the sprite's image definition).
*   **`rect_animation`**: `fireball` (Specifies the animation to use from the `image_file`).
