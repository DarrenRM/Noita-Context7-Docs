# Lightning Thunderburst Projectile

This document details the configuration for the `lightning_thunderburst.xml` projectile entity in Noita, designed for AI-assisted modding.

## Entity Definition

The core of this projectile is based on `data/entities/base_projectile.xml`.

```xml
<Entity name="$projectile_default">
  <Base file="data/entities/base_projectile.xml" />
  <!-- ... other components ... -->
</Entity>
```

## Projectile Component

This component defines the fundamental properties of the projectile.

```xml
<ProjectileComponent
  _enabled="1"
  on_collision_die="0"
  on_death_explode="0"
  on_death_gfx_leave_sprite="0"
  on_lifetime_out_explode="0"
  explosion_dont_damage_shooter="1"
  lifetime="2"
  projectile_type="LIGHTNING"
  muzzle_flash_file="data/entities/particles/muzzle_flashes/muzzle_flash_laser.xml"
  damage="0"
>
  <damage_by_type electricity="1.0" />
</ProjectileComponent>
```

**Key Attributes:**

*   `lifetime`: The duration the projectile exists before expiring (2 seconds).
*   `projectile_type`: Set to `LIGHTNING`, indicating its electrical nature.
*   `damage_by_type electricity`: Specifies that this projectile deals 1.0 damage of type `electricity`.

## Lightning Component

This component specifically handles the electrical and explosive aspects of the projectile.

```xml
<LightningComponent
  is_projectile="1"
  explosion_type="1"
>
  <config_explosion
    never_cache="1"
    camera_shake="7.5"
    explosion_radius="5"
    explosion_sprite="data/particles/explosion_040_electric.xml"
    explosion_sprite_lifetime="0"
    create_cell_probability="1"
    hole_destroy_liquid="0"
    spark_material="spark_electric"
    particle_effect="1"
    damage="0"
    damage_mortals="1"
    physics_explosion_power.min="2"
    physics_explosion_power.max="3"
    physics_throw_enabled="1"
    shake_vegetation="1"
    sparks_count_min="1"
    sparks_count_max="10"
    sparks_enabled="1"
    stains_enabled="1"
    stains_radius="8"
    hole_enabled="1"
    hole_image="data/temp/explosion_hole.png"
  />
</LightningComponent>
```

**Key Attributes within `config_explosion`:**

*   `camera_shake`: The intensity of camera shake upon explosion (7.5).
*   `explosion_radius`: The radius of the explosion effect (5 units).
*   `explosion_sprite`: The visual effect used for the explosion (`data/particles/explosion_040_electric.xml`).
*   `spark_material`: The material used for sparks generated by the explosion (`spark_electric`).
*   `damage_mortals`: Set to `1`, meaning the explosion damages living entities.
*   `physics_explosion_power`: Defines the minimum (2) and maximum (3) force of the physics-based explosion.
*   `sparks_count_min`/`sparks_count_max`: The range for the number of sparks created (1-10).
*   `stains_radius`: The radius of stains left by the explosion (8 units).
*   `hole_enabled`: Set to `1`, indicating that the explosion can create holes.

## Variable Storage

This component stores a reference to the projectile's own XML file.

<VariableStorageComponent
  name="projectile_file"
  value_string="data/entities/projectiles/lightning_thunderburst.xml"
/>
