# Worm Shot Projectile

This document details the configuration of the `worm_shot.xml` entity, which defines the behavior and properties of the worm shot projectile in Noita.

## Core Projectile Properties

The `ProjectileComponent` is central to defining how the projectile behaves in the game world.

### Key Attributes:

| Attribute                 | Description                                                                                                |
| :------------------------ | :--------------------------------------------------------------------------------------------------------- |
| `lob_min`, `lob_max`      | Controls the minimum and maximum lobbing angle of the projectile.                                            |
| `speed_min`, `speed_max`  | Defines the range of initial speeds for the projectile.                                                    |
| `collide_with_world`      | Set to `0`, meaning it does not collide with the static world geometry.                                    |
| `on_death_explode`        | If `1`, the projectile explodes when it dies.                                                              |
| `on_lifetime_out_explode` | If `1`, the projectile explodes when its lifetime expires.                                                 |
| `lifetime`                | The base duration of the projectile in frames.                                                             |
| `lifetime_randomness`     | Adds a random variation to the projectile's lifetime.                                                      |
| `damage`                  | The base damage dealt by the projectile.                                                                   |
| `knockback_force`         | The force applied to entities upon collision.                                                              |
| `penetrate_entities`      | If `1`, the projectile can pass through multiple entities.                                                 |
| `bounces_left`            | The number of times the projectile can bounce off surfaces.                                                |
| `bounce_always`           | If `1`, the projectile will always attempt to bounce rather than passing through.                          |
| `friendly_fire`           | If `1`, the projectile can damage the player who fired it.                                                 |
| `camera_shake_when_shot`  | The intensity of camera shake when the projectile is fired.                                                |
| `muzzle_flash_file`       | Specifies the particle effect to use for the muzzle flash.                                                 |
| `collide_with_shooter_frames` | Number of frames after firing that the projectile will collide with the shooter.                           |

### Explosion Configuration (`config_explosion`):

| Attribute             | Description                                                                                             |
| :-------------------- | :------------------------------------------------------------------------------------------------------ |
| `damage`              | Damage dealt by the explosion. Set to `0` here, likely relying on `AreaDamageComponent`.                |
| `camera_shake`        | Camera shake intensity of the explosion. Set to `0`.                                                    |
| `explosion_radius`    | The radius of the explosion effect.                                                                     |
| `hole_enabled`        | If `1`, the explosion creates holes in terrain.                                                         |
| `sparks_count_max`    | Maximum number of sparks generated by the explosion.                                                    |
| `sparks_count_min`    | Minimum number of sparks generated by the explosion.                                                    |
| `sparks_enabled`      | If `1`, sparks are generated by the explosion.                                                          |
| `spark_material`      | The material used for the sparks (e.g., `blood_worm`).                                                  |

## Homing Behavior

The `HomingComponent` enables the projectile to track and pursue targets.

### Key Attributes:

| Attribute                   | Description                                                                                             |
| :-------------------------- | :------------------------------------------------------------------------------------------------------ |
| `_enabled`                  | Controls whether the homing behavior is active.                                                         |
| `target_who_shot`           | If `1`, the projectile targets the entity that fired it.                                                |
| `homing_targeting_coeff`    | Coefficient determining how strongly the projectile turns towards the target.                           |
| `homing_velocity_multiplier`| Multiplier applied to the projectile's velocity when homing.                                            |
| `detect_distance`           | The maximum distance at which the projectile can detect a target.                                       |
| `just_rotate_towards_target`| If `1`, the projectile only rotates towards the target without changing its forward velocity direction. |
| `max_turn_rate`             | The maximum rate at which the projectile can turn.                                                      |

## Area Damage

The `AreaDamageComponent` applies damage over time to entities within a specified radius.

### First Area Damage Component (Smaller Radius):

| Attribute             | Description                                                                                             |
| :-------------------- | :------------------------------------------------------------------------------------------------------ |
| `aabb_min.x`, `y`     | Minimum X and Y coordinates of the damage area's bounding box.                                          |
| `aabb_max.x`, `y`     | Maximum X and Y coordinates of the damage area's bounding box.                                          |
| `damage_per_frame`    | Damage dealt per frame to entities within the area.                                                     |
| `update_every_n_frame`| How often the damage check is performed.                                                                |
| `entities_with_tag`   | Applies damage to entities possessing this tag (e.g., `hittable`).                                      |
| `damage_type`         | The type of damage dealt (e.g., `DAMAGE_MELEE`).                                                        |
| `death_cause`         | The game's internal identifier for the cause of death (e.g., `$action_worm_shot`).                      |

### Second Area Damage Component (Larger Radius):

This component has a larger `aabb` and potentially different damage values, affecting a wider area or different entity types.
