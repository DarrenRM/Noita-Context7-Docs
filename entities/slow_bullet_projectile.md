# Slow Bullet Projectile

This document details the `bullet_slow.xml` entity, representing a slow-moving projectile often found in Noita decks. It focuses on key attributes that define its behavior, appearance, and effects.

## Core Projectile Properties

The `ProjectileComponent` is central to defining the projectile's behavior.

### Key Attributes:

*   **`lob_min`, `lob_max`**: Controls the minimum and maximum lob angle (in radians) for the projectile.
*   **`speed_min`, `speed_max`**: Defines the range of initial projectile speeds.
*   **`friction`**: How much the projectile slows down over time.
*   **`direction_random_rad`**: Adds a small random deviation to the projectile's initial direction.
*   **`on_death_explode`**: If `1`, the projectile explodes upon death.
*   **`on_lifetime_out_explode`**: If `1`, the projectile explodes when its lifetime expires.
*   **`on_collision_die`**: If `1`, the projectile is destroyed upon collision.
*   **`lifetime`**: The base lifetime of the projectile in frames.
*   **`lifetime_randomness`**: Adds random variation to the projectile's lifetime.
*   **`damage`**: The base damage dealt by the projectile.
*   **`knockback_force`**: The force applied to entities when hit.
*   **`camera_shake_when_shot`**: The amount of camera shake when this projectile is fired.
*   **`muzzle_flash_file`**: Specifies the particle effect for the muzzle flash.
*   **`shoot_light_flash_r`, `shoot_light_flash_g`, `shoot_light_flash_b`, `shoot_light_flash_radius`**: Defines the color and radius of the light flash when the projectile is shot.

### Explosion Configuration (`config_explosion`)

This nested element defines the properties of the explosion when the projectile detonates.

*   **`damage`**: Damage dealt by the explosion.
*   **`camera_shake`**: Camera shake caused by the explosion.
*   **`explosion_radius`**: The radius of the explosion.
*   **`explosion_sprite`**: The particle effect used for the explosion.
*   **`hole_enabled`**: If `1`, the explosion creates holes in terrain.
*   **`ray_energy`**: The energy of the explosion's rays.
*   **`physics_explosion_power.min`, `physics_explosion_power.max`**: The minimum and maximum power of the physics impulse from the explosion.
*   **`physics_throw_enabled`**: If `1`, the explosion can throw physics objects.
*   **`sparks_enabled`**: If `1`, sparks are generated by the explosion.
*   **`stains_enabled`**: If `1`, impact stains are created.
*   **`spark_material`**: The material of the sparks generated.

## Visuals

### Sprite Component

This component defines the visual appearance of the projectile.

*   **`image_file`**: The path to the sprite image.
*   **`additive`**: If `1`, the sprite uses additive blending.
*   **`rect_animation`**: Specifies the name of the rectangle animation to use.

### Particle Emitter Component

This component controls the emission of particles from the projectile.

*   **`emitted_material_name`**: The material of the particles to emit.
*   **`gravity.y`**: The gravity applied to emitted particles.
*   **`x_vel_min`, `x_vel_max`, `y_vel_min`, `y_vel_max`**: The velocity range for emitted particles.
*   **`count_min`, `count_max`**: The number of particles emitted per burst.
*   **`lifetime_min`, `lifetime_max`**: The lifetime range of emitted particles.
*   **`create_real_particles`**: If `1`, creates actual physics particles.
*   **`emit_cosmetic_particles`**: If `1`, emits cosmetic particles that don't interact with physics.
*   **`fade_based_on_lifetime`**: If `1`, particles fade out as their lifetime decreases.

## Other Components

### Light Component

Adds a light source to the projectile.

*   **`radius`**: The radius of the light.
*   **`r`, `g`, `b`**: The RGB color values of the light.

### Audio Component

Handles the sound effects associated with the projectile.

*   **`file`**: The path to the audio bank.
*   **`event_root`**: The root event name for the projectile's sounds.

### Variable Storage Component

Stores custom variables.

*   **`name`**: The name of the variable.
*   **`value_string`**: The string value of the variable. In this case, it stores the projectile's own file path.
