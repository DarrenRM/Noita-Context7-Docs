# Modding: Component Update Order

Understanding the order in which game components are updated is crucial for modders. Many components have associated "systems" that execute update routines every frame. These routines read component values, check for other components, and modify the game world, components, or entities. The sequence of these updates can significantly impact mod behavior, especially when components interact with each other. Some components are purely data holders or have their behavior managed by other systems, meaning they don't have their own independent update routines.
