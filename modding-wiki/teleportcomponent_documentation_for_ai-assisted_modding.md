# TeleportComponent Documentation for AI-Assisted Modding

This documentation details the `TeleportComponent` in Noita, a crucial component for entities that can teleport. Understanding its properties and how to configure them is essential for modders looking to create custom teleportation mechanics, modify existing ones, or implement unique entity behaviors. This guide breaks down the component's parameters and provides examples for effective AI-assisted modding.

## TeleportComponent Properties

The `TeleportComponent` governs the teleportation behavior of an entity. It has several configurable properties that determine how and where the entity teleports.

### Core Properties

| Property Name | Type | Description | Default Value |
|---|---|---|---|
| `teleport_distance` | float | The maximum distance the entity can teleport in a single jump. | 100.0 |
| `teleport_cooldown` | float | The time in seconds before the entity can teleport again. | 2.0 |
| `teleport_chance` | float | The probability (0.0 to 1.0) that the entity will attempt to teleport when the conditions are met. | 1.0 |
| `teleport_target_bias` | float | Influences the direction of teleportation. Higher values bias towards teleporting towards the player. | 0.0 |
| `teleport_target_player_only` | bool | If true, the entity will only attempt to teleport towards the player. | false |
| `teleport_to_ground` | bool | If true, the entity will attempt to teleport to a valid ground position. | false |
| `teleport_to_random_ground` | bool | If true, the entity will teleport to a random valid ground position within range. | false |
| `teleport_to_random_position` | bool | If true, the entity will teleport to a random position within range, not necessarily on the ground. | false |
| `teleport_to_player_position` | bool | If true, the entity will attempt to teleport directly to the player's current position. | false |
| `teleport_to_player_nearby` | bool | If true, the entity will attempt to teleport to a random position near the player. | false |

## Related Components and Concepts

*   **AI Components:** The `TeleportComponent` is often used in conjunction with AI components that dictate when and why an entity should teleport. For example, an AI might trigger a teleport when the entity is low on health or when the player gets too close.
*   **Entity XML:** All component configurations are defined within an entity's XML file. You'll typically find `TeleportComponent` within the `components` tag of an entity definition.
*   **Lua Scripting:** For more complex teleportation logic or custom triggers, you can interact with the `TeleportComponent` using Lua scripting. This allows for dynamic control over teleportation behavior.
