# MoveToSurfaceOnCreateComponent

This documentation page explains the `MoveToSurfaceOnCreateComponent` in Noita, a component that allows entities to be moved to the surface upon their creation. Understanding this component is crucial for modders who wish to control the initial placement and behavior of custom entities within the game world, particularly for those intended to appear at the start of a new run or in specific surface-level scenarios.

## MoveToSurfaceOnCreateComponent

The `MoveToSurfaceOnCreateComponent` is a simple component that, when added to an entity, will cause that entity to be moved to the surface of the current biome when the entity is created. This is often used for entities that should appear at the beginning of a run or in specific surface-level contexts.

### Component Properties

This component does not have any configurable properties. Its functionality is purely to trigger the movement to the surface upon creation.

To use this component, you simply need to add it to an entity definition in your mod's XML files.
