# GameLogComponent Documentation

This document details the `GameLogComponent` in Noita, a crucial component for managing and displaying in-game messages and events. Understanding this component is vital for modders who wish to create custom notifications, track game state, or integrate their mods with the game's logging system.

## Overview of GameLogComponent

The `GameLogComponent` is responsible for handling all text-based messages that appear in the game's log. This includes everything from enemy spawns and item pickups to spell casting and status effects. Modders can interact with this component to add their own custom log entries, control the visibility of existing messages, or even modify how messages are displayed.

## Core Functionality and Usage

The `GameLogComponent` primarily operates through XML definitions within entity files and can be influenced by Lua scripting.
