# ConfigDrugFx Documentation

This documentation explains the `ConfigDrugFx` component in Noita, which is crucial for defining and controlling the visual and gameplay effects of "drugs" or status effects applied to entities. Understanding `ConfigDrugFx` is essential for modders who wish to create custom status effects, modify existing ones, or implement unique gameplay mechanics tied to entity states.

## Understanding ConfigDrugFx

The `ConfigDrugFx` component is an XML-based configuration that dictates how a specific "drug" or status effect manifests within the game. It allows for a wide range of customizations, from simple visual changes to complex behavioral modifications.

### Core Concepts

*   **Drugs:** In Noita's context, "drugs" refer to status effects that can be applied to entities, such as the player, enemies, or even environmental objects. These effects can alter their appearance, movement, damage, or other properties.
*   **ConfigDrugFx:** This is the primary XML component used to define the parameters of these status effects. It's typically found within entity prefabs or applied dynamically through Lua scripting.
