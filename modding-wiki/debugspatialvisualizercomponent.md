# DebugSpatialVisualizerComponent

This documentation covers the `DebugSpatialVisualizerComponent` in Noita, a powerful tool for modders to visualize and debug spatial data within the game. Understanding and utilizing this component is crucial for identifying and resolving issues related to entity placement, collision, and other spatial interactions in your mods.

## Overview

The `DebugSpatialVisualizerComponent` is a built-in Noita component that allows for the visual debugging of various spatial properties of game entities. It's primarily used during development to inspect how entities interact with the game world, their bounding boxes, collision shapes, and other spatial data. While not intended for use in a final release build, it's an invaluable tool for understanding and troubleshooting modded content.

## Usage

To use the `DebugSpatialVisualizerComponent`, you typically need to add it to an entity in your mod. This is usually done within an entity's XML definition. Once added, you can enable specific visualization modes to see different aspects of the entity's spatial data.
