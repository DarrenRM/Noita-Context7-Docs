# LoadEntitiesComponent

This documentation explains the `LoadEntitiesComponent` in Noita, a crucial component for dynamically loading entities into the game world. Understanding this component is vital for modders who wish to introduce custom entities, modify existing ones, or create complex spawning behaviors. It details how entities are defined, referenced, and loaded, providing the necessary information to effectively use this powerful tool in your mods.

## Overview of LoadEntitiesComponent

The `LoadEntitiesComponent` is a fundamental part of Noita's entity system, responsible for managing and instantiating entities based on specific conditions or triggers. It allows for the definition of entities that can be loaded into the game world, often used for things like biome decorations, enemy spawns, or special events.

### Core Functionality

The primary purpose of `LoadEntitiesComponent` is to define a set of entities that can be loaded. This is typically done within an entity's XML definition. When the game encounters an entity with this component, it processes the defined entities for loading.

### Key Concepts

*   **Entity Definitions:** Entities are defined in XML files and can be referenced by their unique IDs.
*   **Loading Conditions:** Entities can be loaded based on various conditions, such as biome, proximity, or specific game states.
*   **Entity Prefabs:** The component often points to prefabs (pre-defined entity configurations) that are then instantiated.

## Using LoadEntitiesComponent in Mods

Modders can leverage `LoadEntitiesComponent` to add custom content or alter game mechanics. This involves defining new entities or modifying existing ones to include this component with custom entity references.

### Defining Entities to Load

Entities that are intended to be loaded by `LoadEntitiesComponent` are typically defined in separate XML files and then referenced by their `id`.
