# StatusEffectDataComponent

This documentation page details the `StatusEffectDataComponent` in Noita, a crucial component for defining and managing status effects within the game. Understanding this component is essential for modders looking to create custom status effects, modify existing ones, or implement unique gameplay mechanics tied to character states.

## Overview of Status Effects

Status effects in Noita are temporary or permanent conditions that alter a character's behavior, stats, or appearance. They are fundamental to many gameplay mechanics, from combat buffs and debuffs to environmental hazards and unique spell interactions. The `StatusEffectDataComponent` is the primary data structure used by the game engine to define these effects.

## `StatusEffectDataComponent` Properties

The `StatusEffectDataComponent` is an XML component that defines various aspects of a status effect. Here are its key properties:

### Core Properties

*   **`effect_id`**: A unique identifier for the status effect. This is often a string.
*   **`effect_name`**: A human-readable name for the status effect.
*   **`effect_description`**: A brief description of what the status effect does.
*   **`is_bad`**: A boolean value (`true` or `false`) indicating whether the effect is considered negative for the affected entity.
*   **`is_permanent`**: A boolean value (`true` or `false`) indicating if the effect does not expire on its own.
*   **`is_hidden_in_ui`**: A boolean value (`true` or `false`) to control if the status effect is displayed in the UI.
*   **`is_stackable`**: A boolean value (`true` or `false`) to determine if multiple instances of the same status effect can be applied and stack.
*   **`max_stack_count`**: If `is_stackable` is `true`, this defines the maximum number of stacks allowed.
*   **`stack_duration_increase`**: If `is_stackable` is `true`, this determines how much the duration of the effect is extended with each new stack.
*   **`stack_effect_multiplier`**: If `is_stackable` is `true`, this defines a multiplier applied to the effect's power or magnitude for each stack.
*   **`stack_visual_effect_multiplier`**: If `is_stackable` is `true`, this defines a multiplier applied to the visual effect's intensity for each stack.
*   **`stack_sound_effect_multiplier`**: If `is_stackable` is `true`, this defines a multiplier applied to the sound effect's volume or intensity for each stack.

### Duration and Timers
