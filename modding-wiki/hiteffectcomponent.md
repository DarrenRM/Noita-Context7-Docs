# HitEffectComponent

This documentation covers the `HitEffectComponent` in Noita, a crucial component for defining visual and auditory feedback when an entity is hit. Understanding and modifying this component is essential for creating immersive and impactful combat experiences in your mods.

## Overview

The `HitEffectComponent` dictates what happens visually and audibly when an entity receives damage. This includes spawning particles, playing sounds, and applying visual effects to the entity itself. By configuring this component, modders can customize the feedback for various damage types or create unique hit reactions for custom entities.

## Component Properties

The `HitEffectComponent` has several properties that can be configured within an entity's XML definition.

### `hit_effect_material`

This property specifies the material used for the hit effect. It often refers to a material defined elsewhere in the game's assets.

### `hit_effect_material_material_type`

This property determines the type of material being used for the hit effect. Common values include:

*   `MATERIAL_TYPE_DEFAULT`
*   `MATERIAL_TYPE_LIQUID`
*   `MATERIAL_TYPE_GAS`
*   `MATERIAL_TYPE_MAGIC`
*   `MATERIAL_TYPE_DEBUG`

### `hit_effect_material_material_type_override`

Allows overriding the default material type for specific hit effects.

### `hit_effect_material_material_type_override_material_type`

The specific material type to override with.
