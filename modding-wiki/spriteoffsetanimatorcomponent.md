# SpriteOffsetAnimatorComponent

This documentation explains the `SpriteOffsetAnimatorComponent` in Noita, a component used to animate the sprite offset of an entity over time. Understanding this component is crucial for modders who wish to create dynamic visual effects for entities, such as pulsating, wobbling, or moving sprites.

## Overview

The `SpriteOffsetAnimatorComponent` allows for the animation of an entity's sprite's offset. This means you can programmatically change the position of the sprite relative to the entity's origin, creating a variety of visual effects. This is achieved by defining keyframes that specify the offset at different points in time.

## Keyframes

The animation is defined by a list of keyframes. Each keyframe specifies a `time` and an `offset`. The `offset` is a pair of `x` and `y` values representing the displacement from the entity's origin.

### Keyframe Structure

Each keyframe is defined within `<keyframe>` tags and has the following attributes:

*   **time**: A floating-point number representing the time in seconds at which this offset should be applied.
*   **x**: A floating-point number representing the horizontal offset.
*   **y**: A floating-point number representing the vertical offset.

### Example Keyframes

```xml
<SpriteOffsetAnimatorComponent
    _tags="animation_component"
    >
    <keyframe time="0.0" x="0" y="0" />
    <keyframe time="0.25" x="5" y="0" />
    <keyframe time="0.5" x="0" y="0" />
    <keyframe time="0.75" x="-5" y="0" />
    <keyframe time="1.0" x="0" y="0" />
</SpriteOffsetAnimatorComponent>
```

This example defines an animation that moves the sprite 5 pixels to the right, then back to the center, then 5 pixels to the left, and finally back to the center over a duration of 1 second.

## Usage in Mods

To use this component in your mod, you would typically add it to an entity's definition within its XML file.

### Example Entity with SpriteOffsetAnimatorComponent

```xml
<entity name="my_animated_entity" >
    <SpriteComponent
        image_file="data/sprites/my_sprite.png"
        offset_x="0"
        offset_y="0"
        >
    </SpriteComponent>
    <SpriteOffsetAnimatorComponent
        _tags="animation_component"
        >
        <keyframe time="0.0" x="0" y="0" />
        <keyframe time="0.5" x="10" y="0" />
        <keyframe time="1.0" x="0" y="0" />
    </SpriteOffsetAnimatorComponent>
    <!-- Other components as needed -->
</entity>
```

In this example, `my_animated_entity` has a `SpriteComponent` and the `SpriteOffsetAnimatorComponent`. The animation will cause the sprite defined in `SpriteComponent` to move horizontally.

## Important Notes

*   The `time` values in keyframes should generally be increasing to ensure a smooth animation.
*   The `SpriteOffsetAnimatorComponent` works in conjunction with the `SpriteComponent`. The offsets defined in the animator are applied *on top of* the base offset defined in the `SpriteComponent`.
*   You can use this component to create a wide range of visual effects, from subtle movements to more complex animations.
*   For more advanced animation control or dynamic behavior, you might need to combine this component with Lua scripting. Refer to the [Lua API](https://noita.wiki.gg/wiki/Modding:_Lua_API) for more information on scripting possibilities.
