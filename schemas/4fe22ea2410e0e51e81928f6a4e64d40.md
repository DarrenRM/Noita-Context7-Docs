# 4fe22ea2410e0e51e81928f6a4e64d40

This XML file defines the schema for various game components in Noita. It acts as a blueprint, detailing the structure and types of data that each component can hold. These components are fundamental building blocks for entities within the game, dictating their behavior, properties, and interactions. By defining these schemas, Noita can consistently load and interpret the data for its vast array of enemies, spells, environmental elements, and other game objects.

## File Structure

The file is structured as a collection of `<Component>` elements, each representing a distinct game component. The root element is `<Schema>`, which contains all the `<Component>` definitions.

Each `<Component>` element has a `component_name` attribute that uniquely identifies the component (e.g., `AIAttackComponent`, `AIComponent`, `CardinalMovementComponent`).

Inside each `<Component>`, there are `<Var>` elements. These `<Var>` elements define individual variables or properties that belong to that component. Each `<Var>` element has the following attributes:

*   `name`: The name of the variable.
*   `size`: The size of the variable in bytes. This can indicate the data type or storage allocation.
*   `type`: The data type of the variable. Common types include `int`, `float`, `bool`, and C++ string types like `class std::basic_string<char,struct std::char_traits<char>,class std::allocator<char> >`. Some types also indicate enumerations, such as `enum RAGDOLL_FX::Enum`.

The `<Var>` elements can be empty, indicating that the variable's value is stored elsewhere or is implicitly defined.

## Key Patterns

*   **Component-Based Architecture:** The file clearly demonstrates Noita's use of a component-based entity system. Each `<Component>` represents a self-contained piece of functionality or data.
*   **Variable Definitions:** The `<Var>` tags are used to define the properties of each component. The `type` attribute is crucial for understanding how the data is stored and interpreted.
*   **String Representation:** C++ string types are explicitly defined, often with a `size` attribute that suggests a fixed buffer size.
*   **Boolean Flags:** `bool` types are used for simple on/off states or flags.
*   **Numerical Properties:** `int` and `float` types are used for numerical values like distances, probabilities, durations, and speeds.
*   **Enum Types:** The presence of `enum` types indicates that certain variables hold values from a predefined set of options.
