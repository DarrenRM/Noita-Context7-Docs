## File Structure

The file is structured as a series of `<Component>` elements, each representing a distinct game component.

*   **Root Element:** The entire file is enclosed within a `<Schema>` tag, which has a `hash` attribute.
*   **Component Elements:** Each `<Component>` tag has a `component_name` attribute that identifies the specific game component it describes (e.g., `AIAttackComponent`, `AIComponent`, `ElectricityReceiverComponent`).
*   **Variable Elements:** Inside each `<Component>`, individual properties or variables are defined using `<Var>` tags.
    *   **`name` attribute:** Specifies the name of the variable (e.g., `use_probability`, `min_distance`, `radius`).
    *   **`size` attribute:** Indicates the memory size allocated for the variable.
    *   **`type` attribute:** Defines the data type of the variable (e.g., `int`, `float`, `bool`, `class std::basic_string<char,struct std::char_traits<char>,class std::allocator<char> >` for strings, `class ConfigGun` for custom types).
*   **Nested Structures:** Some `type` attributes, like `struct types::iaabb`, suggest that variables can represent more complex nested data structures.

*   **Component-Based Design:** The game's data is organized around distinct components, allowing for modularity and reusability of functionality.
*   **Variable Definitions:** Each component's behavior is defined by a set of named variables with specified data types and sizes.
*   **Data Types:** A mix of fundamental C++ data types (`int`, `float`, `bool`) and custom class/struct types are used, indicating a complex underlying game engine. String types are explicitly defined as `class std::basic_string<char,struct std::char_traits<char>,class std::allocator<char> >`.
