# c0054a6eca20ac8ebb3a24bbf27061fd

This XML file defines the structure and properties of various game components used within Noita. These components are the building blocks for entities, dictating their behavior, appearance, and physical interactions. This schema file acts as a blueprint, specifying the types and names of variables that each component can possess, allowing the game engine to correctly load and interpret entity data. Essentially, it's a data dictionary for game object attributes.

## File Structure

The file is structured as a collection of `<Component>` elements, each representing a distinct type of game component. Each `<Component>` has a `component_name` attribute that uniquely identifies it. Inside each `<Component>`, there are `<Var>` elements, which define individual properties or variables associated with that component.

Each `<Var>` element has the following attributes:
*   `name`: The name of the variable.
*   `size`: The size of the variable in bytes.
*   `type`: The data type of the variable (e.g., `int`, `float`, `bool`, `class std::basic_string<char,struct std::char_traits<char>,class std::allocator<char> >`, `struct ValueRange`, `struct ceng::math::CXForm<float>`, `class ceng::math::CVector2<float>`).

The `type` attribute can sometimes indicate complex data structures or string types, often referencing C++ class names.

## Key Patterns

*   **Component-Based Architecture:** The file clearly demonstrates Noita's use of a component-based entity system. Each `<Component>` defines a specific set of functionalities or attributes that can be attached to game entities.
*   **Variable Definitions:** The `<Var>` tags are consistently used to declare the properties of each component. The `size` and `type` attributes provide crucial information about how this data is stored and interpreted.
*   **Data Types:** A variety of data types are used, ranging from primitive types like `int`, `float`, and `bool` to more complex C++ types like strings (`class std::basic_string...`), custom structs (`struct ValueRange`, `struct ceng::math::CXForm<float>`), and vector types (`class ceng::math::CVector2<float>`).
*   **AI and Physics Focus:** The sampled components (`AIAttackComponent`, `AIComponent`, `ControllerGoombaAIComponent`, `PhysicsPickUpComponent`) suggest a strong emphasis on defining artificial intelligence behaviors and physical interactions for game entities.
*   **Naming Conventions:** Variable names are generally descriptive, indicating their purpose (e.g., `use_probability`, `min_distance`, `auto_turn_around_enabled`, `radius`, `pick_up_strength`).
